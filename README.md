Сервис сокращения ссылок

Что умеет
1. Сокращать ссылки – делает из длинного URL короткий, например http://localhost:8080/AbCdEfGh
2. Уникальность – если два разных человека сокращают одну и ту же ссылку, у них получаются разные короткие версии
3. Лимиты переходов – можно задать, сколько раз по ссылке можно перейти (по умолчанию 10)
4. Время жизни – ссылки сами удаляются через 24 часа (настраивается в конфиге)
5. Уведомления – присылает сообщения, когда лимит исчерпан или ссылка протухла
6. Управление – можно смотреть свои ссылки, менять лимиты, удалять

Работа из консоли – всё через меню

Как запустить
Требования:
1. Java 21
2. Maven (для сборки)

Сборка и запуск:
# Клонируем (если с гита)
git clone https://github.com/ehor4ek/link-shortener.git
cd link-shortener

# Собираем
mvn clean compile

# Запускаем
mvn exec:java -Dexec.mainClass="com.linkshortener.Main"

Тесты
mvn test

Как пользоваться
При запуске появляется меню:

1. Создать короткую ссылку
2. Перейти по короткой ссылке  
3. Мои ссылки
4. Изменить лимит переходов
5. Удалить ссылку
6. Просмотреть уведомления
7. Информация о пользователе
8. Помощь
0. Выход

Пример работы
Выбираем "1. Создать короткую ссылку"

Вводим любой URL, например: https://www.youtube.com/watch?v=dQw4w9WgXcQ

Можно поставить лимит переходов (скажем, 5)

Получаем что-то вроде:

✓ Ссылка успешно создана!
Оригинальный URL: https://www.youtube.com/watch?v=dQw4w9WgXcQ
Короткая ссылка: http://localhost:8080/AbCdEfGh
Код ссылки: AbCdEfGh
Лимит переходов: 5
Истекает: 2024-01-15T14:30:00
Выбираем "2. Перейти по короткой ссылке", вводим код AbCdEfGh

Ссылка откроется в браузере, счётчик переходов увеличится

Особенности реализации:
1. Хранение – всё в памяти, при перезапуске данные сбрасываются (так и задумано)
2. Уникальные коды – генерируются случайно, 8 символов (A-Z, a-z, 0-9)
3. Идентификация – каждому пользователю присваивается UUID при первом запуске
4. Фоновые задачи – раз в час чистит просроченные ссылки
5. Конфиг – настройки в application.properties (можно менять лимиты, время жизни и т.д.)
6. Обработка ошибок – валидация URL, проверка прав доступа, понятные сообщения

Тестирование
Написаны тесты на основные функции:
1. Генерация кодов (уникальность, длина)
2. Работа сервиса ссылок (создание, переходы, лимиты)
3. Интеграционные тесты (несколько пользователей, изоляция)

Запускаются через mvn test.

Конфигурация
Файл application.properties создаётся автоматически при первом запуске. Можно поменять:

properties
# Базовый URL для коротких ссылок
shortlink.base.url=http://localhost:8080/

# Длина короткого кода (от 4 до 16)
shortlink.code.length=8

# Время жизни ссылок в часах
shortlink.ttl.hours=24

# Лимит переходов по умолчанию
default.click.limit=10

Заметки по коду:
1. Пакеты разложены по слоям (core, service, storage, util)
2. Используются синглтоны для сервисов
3. Кастомные исключения для разных ситуаций

Проект соответствует всем пунктам ТЗ, тесты проходят, интерфейс понятный.
